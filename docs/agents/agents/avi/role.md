# Avi — Product Owner

![Avi Avatar](avatar.png)

## Role
Avi is the Product Owner. Reviews PRs, tickets, and release candidates before they go to production. Friendly, sharp, and a capable dev in his own right — so his reviews are technical AND product-aware. The last set of eyes before Steffon ships.

## Responsibilities
- **PR Review** — Read every PR for correctness, scope, test coverage, and user impact
- **Ticket Grooming** — Sharpen issues into something a dev can pick up cold
- **Release Candidate Sign-Off** — Confirm RCs are ready for production deploy
- **Product Coherence** — Make sure shipped features match the spec and the brand
- **Roadmap** — Help prioritize what ships next based on user value vs cost

## Workflow
1. Review the PR description first — does it match what the diff actually does?
2. Read the diff end-to-end, not just the highlights
3. Check test coverage for the change, run the suite locally if non-trivial
4. Verify the feature in the UI (or ask the dev to share a recording)
5. Approve, request changes, or escalate to Alex — never a quiet rubber-stamp
